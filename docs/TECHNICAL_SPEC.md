## 1. Service Map

Two services + one database:

| Service | Platform | Language | Purpose |
|---------|----------|----------|---------|
| sift-web | Vercel (Hobby) | TypeScript (Next.js 15) | Frontend, API routes (DB reads), Clerk auth, SSE streaming |
| sift-api | Railway | Python (FastAPI + LangGraph) | Background pipeline (10-min scheduler), multi-source comparison |
| sift-db | Neon (free tier) | PostgreSQL 17 + pgvector | Shared database, single source of truth |

## 2. Database Schema

```sql
-- Enable pgvector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- ─── Articles ───────────────────────────────────────────
-- One row per article. Populated by the background pipeline.

CREATE TABLE articles (
  id TEXT PRIMARY KEY,                           -- stableHash(source_url + title)
  title TEXT NOT NULL,
  summary TEXT,                                  -- AI-generated by Claude Haiku
  source_url TEXT NOT NULL,
  source_name TEXT NOT NULL,
  image_url TEXT,                                -- from RSS feed, nullable
  category TEXT NOT NULL,                        -- 'top' | 'technology' | 'business' | etc.
  published_date TIMESTAMPTZ,
  embedding VECTOR(1024),                        -- Voyage AI voyage-3-lite
  read_time INTEGER DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT unique_source_url UNIQUE (source_url)
);

-- Query: "give me the latest 7 technology articles"
CREATE INDEX idx_articles_category_date
  ON articles (category, published_date DESC NULLS LAST);

-- Query: "find articles similar to this embedding"
CREATE INDEX idx_articles_embedding
  ON articles USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 50);

-- Query: keyword search
-- (Note: create after initial data load for better index quality)
-- ALTER TABLE articles ADD COLUMN search_vector TSVECTOR
--   GENERATED ALWAYS AS (to_tsvector('english', title || ' ' || COALESCE(summary, ''))) STORED;
-- CREATE INDEX idx_articles_fts ON articles USING GIN (search_vector);


-- ─── Custom Topics ──────────────────────────────────────
-- User-defined feeds. Query embedding matched against article embeddings.

CREATE TABLE custom_topics (
  id TEXT PRIMARY KEY,                           -- generated UUID
  user_id TEXT NOT NULL,                         -- Clerk user ID
  name TEXT NOT NULL,                            -- display name, e.g. "AI in Healthcare"
  query TEXT NOT NULL,                           -- original user input
  embedding VECTOR(1024),                        -- embedded query for similarity search
  created_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT unique_user_topic UNIQUE (user_id, name)
);

CREATE INDEX idx_custom_topics_user
  ON custom_topics (user_id);


-- ─── Bookmarks ──────────────────────────────────────────

CREATE TABLE bookmarks (
  user_id TEXT NOT NULL,                         -- Clerk user ID
  article_id TEXT NOT NULL REFERENCES articles(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),

  PRIMARY KEY (user_id, article_id)
);

CREATE INDEX idx_bookmarks_user
  ON bookmarks (user_id, created_at DESC);


-- ─── Pipeline State ─────────────────────────────────────
-- Tracks last successful refresh per category to avoid redundant work.

CREATE TABLE pipeline_state (
  category TEXT PRIMARY KEY,
  last_refreshed_at TIMESTAMPTZ NOT NULL,
  article_count INTEGER DEFAULT 0,
  error TEXT                                     -- last error if any, NULL if clean
);
```

**Notes:**
- `VECTOR(1024)` assumes Voyage AI `voyage-3-lite` model (1024 dimensions). Adjust if using a different model.
- `ivfflat` index requires ~1000 rows before it's effective. Use exact search (`<=>` operator without index) until then.
- Article `id` is `stableHash(source_url + title)` — same function from current `lib/utils.ts`.

## 3. API Contract — sift-api (Python / Railway)

Base URL: `https://sift-api-production.up.railway.app`

### POST /pipeline/refresh

Triggered by Vercel Cron. Fetches RSS feeds, summarizes with Claude, embeds with Voyage, stores in Postgres.

**Request:**
```json
{
  "categories": ["top", "technology", "business", "science", "energy", "world", "health"],
  "force": false
}
```
- `categories`: which categories to refresh. Omit for all.
- `force`: if true, refresh even if recently updated.

**Response (200):**
```json
{
  "results": {
    "technology": { "new_articles": 3, "skipped": 2, "errors": 0 },
    "business": { "new_articles": 5, "skipped": 0, "errors": 0 },
    ...
  },
  "duration_ms": 4200
}
```

**Response (500):**
```json
{
  "error": "Pipeline failed",
  "details": "Claude API timeout on category: energy",
  "partial_results": { ... }
}
```

**Auth:** API key in `X-Pipeline-Key` header (shared secret between Vercel Cron and Railway).

---

### POST /analyze/compare

User-triggered multi-source comparison via LangGraph.

**Request:**
```json
{
  "topic": "Federal Reserve interest rate decision",
  "sources": ["reuters", "bbc", "associated press"]
}
```
- `sources`: optional, defaults to ["reuters", "bbc", "associated press"]

**Response (200):**
```json
{
  "topic": "Federal Reserve interest rate decision",
  "comparison": "All three outlets agree the Fed held rates steady...",
  "sources_checked": ["reuters", "bbc", "associated press"],
  "claims": [
    {
      "claim": "Fed held rates at 5.25-5.50%",
      "agreement": "unanimous",
      "sources": ["reuters", "bbc", "ap"]
    },
    {
      "claim": "Two rate cuts expected in 2026",
      "agreement": "disputed",
      "sources_for": ["reuters", "bbc"],
      "sources_against": ["ap"]
    }
  ],
  "duration_ms": 8500
}
```

**Auth:** Bearer token (Clerk JWT forwarded from Next.js).

---

### GET /health

**Response (200):**
```json
{
  "status": "ok",
  "version": "1.0.0",
  "db_connected": true,
  "last_pipeline_run": "2026-03-28T14:30:00Z"
}
```

## 4. API Contract — sift-web (Next.js / Vercel)

### GET /api/news?category={category}

Returns articles for a fixed category. Pure database read.

**Response (200):**
```json
{
  "articles": [
    {
      "id": "abc123",
      "title": "OpenAI launches new agent framework",
      "summary": "OpenAI announced a new framework for building autonomous agents...",
      "sourceUrl": "https://techcrunch.com/2026/03/28/openai-agents",
      "sourceName": "TechCrunch",
      "imageUrl": "https://techcrunch.com/wp-content/uploads/2026/03/openai.jpg",
      "category": "technology",
      "publishedDate": "2026-03-28T10:00:00Z",
      "readTime": 1
    },
    ...
  ],
  "count": 7,
  "lastRefreshed": "2026-03-28T14:30:00Z"
}
```

**Latency target:** <50ms (Postgres query + JSON serialization)

---

### GET /api/news/topic?q={query}&userId={clerkId}

Returns articles for a custom topic via vector similarity search.

**Response (200):**
```json
{
  "articles": [ ... ],
  "count": 5,
  "matchQuality": "strong",
  "fallbackUsed": false
}
```

If `matchQuality` is `"weak"` (< 3 results above similarity threshold), the endpoint triggers a Claude web_search fallback and returns `"fallbackUsed": true`. Fallback results are also stored in Postgres for future queries.

---

### POST /api/compare

Proxies to the Python service's `/analyze/compare` endpoint.

**Request/Response:** Same as Python service contract above.

---

### GET /api/cron/refresh

Called by Vercel Cron. Triggers the pipeline on Railway.

**Auth:** Vercel Cron sends a `CRON_SECRET` header. The route verifies it before calling Railway.

**Response (200):**
```json
{ "triggered": true, "timestamp": "2026-03-28T14:30:00Z" }
```

---

### SSE: GET /api/news/stream?category={category}

Server-sent events endpoint for streaming article delivery on cache miss.

**Event format:**
```
event: article
data: {"id":"abc","title":"...","summary":"...","sourceUrl":"...", ...}

event: article
data: {"id":"def","title":"...","summary":"...","sourceUrl":"...", ...}

event: done
data: {"count": 5, "cached": false}
```

**When used:** Only on cold loads where Postgres has no articles for the requested category (rare — pipeline keeps it fresh). Client falls back to regular GET if SSE is unavailable.

**Image extraction from RSS:**
Most RSS feeds include images via one of these tags:
- `<enclosure url="..." type="image/jpeg" />`
- `<media:content url="..." medium="image" />`
- `<media:thumbnail url="..." />`
- `<image><url>...</url></image>` (channel-level logo, skip this)

The pipeline should check all three article-level tags and use the first valid one.

**Important:** These feed URLs should be verified before implementation. Some publishers change feed URLs, add authentication, or discontinue feeds. The pipeline should handle feed failures gracefully (log error, skip feed, continue with others).
